# Text Submission Endpoint - Quick Start Guide

## Setup

1. Install dependencies:
```bash
pip install -r requirements.txt
```

2. Run the application:
```bash
python app.py
```

The server will start on `http://localhost:5000`

## Usage

### Submit Text for Analysis

**Request:**
```bash
curl -X POST http://localhost:5000/submit \
  -H "Content-Type: application/json" \
  -d '{"text": "Your text to analyze here"}'
```

**Response:**
```json
{
  "success": true,
  "result": {
    "label": "Your text is Human written",
    "confidence": 0.1234,
    "confidence_percentage": 12,
    "reasoning": "Transparency label placeholder - implementation pending."
  },
  "signals": {
    "llm_judgment": {
      "score": 0.5,
      "reasoning": "LLM judgment placeholder - implementation pending."
    },
    "stylometry": {
      "score": 0.5,
      "reasoning": "Stylometry placeholder - implementation pending."
    },
    "perplexity": {
      "score": 0.5,
      "reasoning": "Perplexity placeholder - implementation pending."
    }
  },
  "status_code": 200
}
```

### Health Check

**Request:**
```bash
curl http://localhost:5000/health
```

**Response:**
```json
{
  "status": "healthy"
}
```

## Rate Limiting

The endpoint enforces a rate limit of **10 requests per minute per IP address**.

When the limit is exceeded, you'll receive:
```json
{
  "error": "Rate limit exceeded",
  "message": "Maximum 10 requests per minute allowed. Please try again later.",
  "status_code": 429
}
```

## Validation Rules

- `text` field is required
- `text` must be a non-empty string
- `text` cannot contain only whitespace
- `text` maximum length is 1,000,000 characters
- Request must be valid JSON

## Error Responses

**Missing field (400):**
```json
{
  "error": "Validation error",
  "message": "Missing required field: text",
  "status_code": 400
}
```

**Empty text (400):**
```json
{
  "error": "Validation error",
  "message": "Field text cannot be empty.",
  "status_code": 400
}
```

**Rate limit exceeded (429):**
```json
{
  "error": "Rate limit exceeded",
  "message": "Maximum 10 requests per minute allowed. Please try again later.",
  "status_code": 429
}
```

## Architecture

The implementation consists of:

- **app.py**: Flask application and endpoints
- **config.py**: Configuration and constants
- **normalizer.py**: Content preprocessing
- **signals.py**: Placeholder signal detection (LLM, Stylometry, Perplexity)
- **confidence.py**: Confidence scoring and label generation
- **rate_limiter.py**: Custom rate limiting implementation

## Confidence Score Thresholds

| Confidence | Label |
|------------|-------|
| 0.00-0.15 | Your text is Human written |
| 0.15-0.30 | Most of Your text is Human Written |
| 0.30-0.42 | Your text is Most Likely Human written, may include parts generated by AI |
| 0.42-0.50 | Your text is Likely Human written, may include parts generated by AI |
| 0.50-0.70 | Your text contains mixed signals, with some parts generated by AI |
| 0.70-0.80 | Your text is Likely AI-generated |
| 0.80-0.90 | Your text is Most Likely AI-generated |
| 0.90-0.97 | Most of Your text is AI-Generated |
| 0.97-1.00 | Your text is AI-Generated |
